# E5CM-CG

一个使用 `Python + Pygame + OpenCV` 重构街机游戏《e舞成名》流程、界面演出和谱面播放体验的非官方项目。

这个仓库当前更像“街机流程重建工程”，而不是只做截图临摹的 UI demo。它已经把开场、投币、模式选择、选歌、加载、游玩、结算这条主链路基本串起来了，并且开始把大量布局和表现参数迁到 `json/` 配置里。

## 当前状态

按当前仓库代码，主流程已经覆盖：

- 开场幻灯片与菜单背景视频
- 投币场景（内含 1P / 2P 选择）
- 登陆磁卡
- 个人资料
- 大模式选择
- 子模式选择
- 选歌
- 加载页
- 谱面播放器
- 结算

目前真正接到可游玩链路的“大模式”主要是 `花式` 和 `竞速`。`派对`、`DIY`、`WEF` 已有按钮和资源占位，但还没有完整接到后续流程。

## 代码现状概览

从当前仓库结构看，职责已经比较明确：

- `main.py`
  主循环、窗口初始化、全局投币逻辑、系统菜单、全局背景视频、场景切换。
- `scenes/`
  场景实现层。核心链路从 `场景_投币.py` 一直到 `场景_结算.py`。
- `ui/`
  UI 与表现层，包含按钮特效、场景过渡、准备就绪动画、结算布局共享、圆环频谱、谱面渲染器等。
- `core/`
  路径、音频、视频、字体、对局状态、歌曲记录，以及判定 / 计分 / 谱面构建等基础玩法模块。
- `json/`
  布局与运行时数据。既有受版本控制的静态布局，也有本地运行后产生的用户数据。
- `打包专用资源/`
  打包时使用的 `songs/` 和 `json/` 默认资源集。

当前最重的几个模块是：

- `scenes/场景_选歌.py`
- `scenes/场景_谱面播放器.py`
- `ui/谱面渲染器.py`

这说明项目已经把复杂能力集中打通，但也还处在持续整理阶段。仓库内目前没有自动化测试，主要依赖独立调试入口和场景内调试工具验证。

## 功能概览

- 场景系统：统一场景切换、过渡、上下文共享、全局背景视频和音乐管理。
- 投币与开局：支持投币计数、扣费开局、1P / 2P 选择，以及可修改的投币快捷键。
- 歌曲扫描：从 `songs/` 目录扫描 `.sm` 谱面，按大模式 / 子模式组织歌曲列表，并维护歌曲游玩记录与最高分。
- 选歌系统：有独立布局配置、设置页、歌曲信息读取、2P 加入逻辑、历史记录读取。
- 谱面播放器：包含准备就绪动画、谱面解析、判定、计分、血量、暂停菜单、击中特效、背景图 / 背景视频切换。
- 结算系统：支持结算页布局配置、新纪录写回、个人资料经验与段位更新。
- 个人资料：支持昵称、头像、本地图片导入、进度与段位显示。
- 打包链路：提供 PyInstaller 打包脚本和 Inno Setup 安装器脚本。

## 目录结构

```text

├─ 测试入口-选歌.py        # 独立选歌调试入口
```

## 运行环境

推荐环境：

- Windows
- Python `3.10+`
- UTF-8 终端 / 编辑器

安装依赖：

```bash
pip install pygame opencv-python numpy
```

说明：

- `OpenCV` 主要用于开场和背景视频播放；未安装时程序会退化运行并在界面上给出提示。
- `numpy` 主要用于频谱装饰和部分图形处理，建议一并安装。
- 如果你只想打包，再额外安装：

```bash
pip install pyinstaller
```

## 启动方式

运行主程序：

```bash
python main.py
```

默认会从开场幻灯片进入主流程，落到 `投币` 场景。

## 默认按键

- `F`：投币
- `1` / `3` / `5`：模拟左 / 右 / 确认踏板
- `ESC`：系统菜单；在谱面播放中为暂停菜单
- `F11`：全屏切换
- 鼠标左键：常规按钮点击

补充说明：

- `F` 只是默认投币键，启动后可在 `ESC` 系统菜单里改成任意按键。
- 非游玩场景的 `ESC` 菜单目前主要提供：修改投币键、开关背景音乐、退出程序。

## songs 目录约定

项目会从本地 `songs/` 目录扫描 `.sm` 文件。按当前代码，目录结构建议类似：

```text
songs/
├─ 花式/
│  ├─ 学习/
│  ├─ 表演/
│  ├─ 疯狂/
│  ├─ club/
│  ├─ 情侣/
│  └─ 混音/
└─ 竞速/
   ├─ 疯狂/
   ├─ club/
   ├─ 情侣/
   └─ 混音/
```

说明：

- 仓库根目录下的 `songs/` 默认被 `.gitignore` 忽略，适合放本地完整歌曲库。
- `打包专用资源/songs/` 是打包脚本真正会复制进发布目录的歌曲资源。
- 选歌与结算使用 `json/歌曲记录索引.json` 维护游玩次数和最高分。

## json 与本地数据

当前 `json/` 目录同时承担两类数据：

- 受版本控制的布局 / 配置文件
  - `选歌布局.json`
  - `谱面渲染器_布局.json`
  - `结算场景布局.json`
  - `准备就绪动画_设置.json`
- 本地运行时数据
  - `全局设置.json`
  - `选歌设置.json`
  - `加载页.json`
  - `歌曲记录索引.json`
  - `电视跟跳设置.json`
  - `个人资料.json`
  - `json/个人资料/` 下的头像等用户文件

这些运行时文件多数已在 `.gitignore` 中排除。打包默认配置位于 `打包专用资源/json/`。

## 调试入口

当前仓库里最直接的调试入口是：

```bash
python 测试入口-加载页.py
python 测试入口-选歌.py
python 谱面布局调试器.py
```

用途分别是：

- `测试入口-加载页.py`：直接验证加载页场景
- `测试入口-选歌.py`：直接进入选歌场景
- `谱面布局调试器.py`：独立调整谱面渲染布局、血条、头像、频谱等显示

此外，`场景_选歌.py`、`场景_谱面播放器.py`、`ui/谱面渲染器.py` 内部也已经集成了不少调试开关和布局覆盖逻辑。

## 打包

当前打包流程分两步：

1. 生成 onedir 版本

```bash
python windows编译脚本.py
```

2. 如需安装包，再用 Inno Setup 编译：

- 脚本文件：`打包成安装器.iss`
- 批处理：`打包成安装包.bat`

打包脚本会：

- 先用 PyInstaller 构建 `E5CM-CG.exe`
- 再把 `core/`、`scenes/`、`ui/`、`UI-img/`、`冷资源/`、`backmovies/` 整体复制到输出目录
- 并优先复制 `打包专用资源/songs/` 与 `打包专用资源/json/`

## 当前限制

- 代码明显偏向 Windows 本地运行和打包环境。
- 中文文件名、资源路径、字体资源较多，跨平台时需要额外处理。
- `花式` / `竞速` 之外的大模式还没有完整接入。
- 自动化测试尚未建立，当前主要依赖人工跑场景和调试脚本。
- 核心大文件已经很重，后续如果要长期维护，建议继续拆分 `选歌`、`谱面播放器`、`谱面渲染器`。
